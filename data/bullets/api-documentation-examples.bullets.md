- Follow general standards for file documentation.
- For module files (*.module), install files (*.install), and include files (*.inc), adhere to the specific standards.
- For files containing a single class, follow object-oriented coding standards.
- For tpl.php files, adhere to special standards and note that there is no @ingroup themeable in the override.
- Follow documentation standards for functions.
- For generic functions and callback functions, adhere to the specific standards.
- For callback functions that are passed to other functions as arguments, follow the standards for documenting callback functions.
- For hook definition functions, follow the standards for documenting hook definitions.
- For hook implementations, follow the standards for documenting hook implementations.
- For hook_update_N implementation (update function), follow Drupal API documentation standards for functions.
- For form-generating functions (including validate/submit), follow form-generating functions standards.
- For hook_menu() callbacks, follow standards for documenting hook_menu() callbacks.
- For render API callback functions, follow Render API callback functions standards.
- For theme_foo() themeable function, follow standards for documenting themeable functions.
- Follow standards for documenting classes and interfaces.
- For classes with or without a namespace, interfaces, member functions, member constants, and member variables, adhere to the specific standards.
- For plugin annotation, follow standards for plugin annotation.
- For constants, follow the standards for constants.
- For global variables, document them in separate api.php files.
- For bullet lists, follow standards for using lists.
- For sections, sub-sections, and in-page references, follow standards for using sections.
- For code samples (@code), follow standards for using @code.
- For @link, follow standards for using @link.
- For @see, follow standards for using @see.
- For defining a topic/group, follow standards for using @defgroup, @ingroup, and @addtogroup.
- For @ingroup and @addtogroup, follow standards for using @defgroup, @ingroup, and @addtogroup.
